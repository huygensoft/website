# Locker App Version 2 Enhancements

## Temporary Unlock Codes
- Plain text QR Codes that contain a timestamp and locker door to open.
- Using TOTP and AES256 to encode the string

## UI Updates Needed
- No indication or message that network/API/Salesforce outage.
- No changes or additional screens.
- When a Temporary Unlock Code is scanned
-- Do not allow the text of the TUC to be displayed in the access code box
-- Or have it cleared quickly as soon as it is detected to be a TUC
-- Change to the Success screen and ‘remember’ this door for as long as the screen is displayed
--- For the Reopen button.
--- Locker App forgets the locker as soon as the Success screen is changed/replaced.
--- !! This should not affect the normal access code process, that would check with the API each time the Reopen button is pushed.

## Encoded QR Code String
- Plain Text String: ‘TUC|{timestamp}|{LockerDoorNumber}’
-- TUC ← Indicator that QR Code is a Temporary Unlock Code
-- Timestamp: 1622179531569 or "2021-05-28T05:25:38.333Z"
-- LockDoorNumber
- Encoded with AES256
-- Secret Key and IV (Initialization vector) / Salt
-- Shared with Locker App and API

## Processing Temporary Unlock Code & Opening Locker
- Decode TUC string, using Secret Key
- IF successfully decrypted
-- Check timestamp in code against current time
--- IF within X minutes of original timestamp, open the locker specified in the TUC
--- ELSE do nothing, show nothing.

## Configuration Variables
- Variables stored in Configuration.ini
-- Temporary Unlock Code original timestamp timeout period in minutes
-- Secret or Salt or Private Key
--- Better place to store this?
--- Would be nice to update/change this often without having to recompile, etc.

# Locker Monitoring
Simple implementation of monitoring the lockers

## Monitors
- Locker door left open for X minutes
- Temperature goes above X degrees

## Sending Alerts
Request
- Route: https://api.onqpm.com/v1/locker/alert
- Type: POST
- Headers:
-- Api-Key: 89bb7d3a-ac37-43fe-98ff-2340213e82fa
- Body (JSON):
-- message (string, required)

Responses
{
"result": "success",
}

## Configuration Variables
- Variables stored in Configuration.ini
-- Alert Door Open in minutes
-- Alert High Temperature in degrees Celsius less
